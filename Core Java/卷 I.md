## Ch04   对象与类

OOP：规模较大

- 对象的行为

- 对象的状态

- 对象标识

  如何辨别具有相同行为与状态的不同对象

类（class）构造（construct）<u>对象</u>的过程称为创建类的实例（instance）

封装（encapsulation）

对象中的数据：实例域（instance field） 		——> 状态

操纵数据的过程：方法（method）

**实现封装的关键**：绝对不能让类中的方法**直接**的访问其他类的实例域，程序仅通过对象的方法与数据对象进行交互



编写OOP：首先从设计类开始，然后再往每个类中添加方法。识别类的简单规则是在分析问题的过程中寻找名词， 而方法对应着动词。
